## 1. 프로젝트 개요

### 1.1 프로젝트 설명
**StudyCoach**는 학원 및 교습소를 위한 학생 성과 관리 SaaS 플랫폼입니다. 
헬스장의 바디코디 프로그램처럼, 학생의 학습 데이터를 시각화하여 
성적 추이, 목표 달성도, 출석률 등을 직관적인 그래프와 대시보드로 제공합니다.

### 1.2 핵심 가치 제안 (Value Proposition)
- 📊 데이터 시각화: 학생의 성적, 목표 달성도를 그래프로 한눈에 파악
- 🎯 목표 기반 관리: 개인별 맞춤 목표 설정 및 달성도 추적
- 📈 성과 추적: 시간별 성장 곡선으로 학습 효과 입증
- 💪 동기부여: 바디코디처럼 시각적 피드백으로 학생 동기부여
- 🔄 실시간 업데이트: 학부모와 학생이 실시간으로 성과 확인

### 1.3 비즈니스 모델
- B2B SaaS: 학원/교습소 대상 구독형 서비스
- Freemium: 무료 체험 → 유료 구독 전환
- 월/연 구독제: 학생 수 기반 가격 책정

## 4. 핵심 기능 명세

### 4.1 MVP 핵심 기능 (Phase 1)

#### 1. 학생 정보 관리
기능 설명:
- 학생 등록 (이름, 학년, 연락처, 목표 점수 등)
- 학생 프로필 관리
- 학생 검색 및 필터링

상세 요구사항:
- 학생 등록 폼
  - 필수: 이름, 생년월일, 학년
  - 선택: 연락처, 학부모 정보, 목표 점수, 약점 과목
  
- 학생 목록 뷰
  - 카드형/리스트형 전환 가능
  - 검색: 이름, 학년별 필터
  - 정렬: 이름순, 성적순, 등록일순

#### 2. 성적 데이터 입력
기능 설명:
- 시험 점수 입력
- 과제 점수 입력
- 출석률 자동 계산

상세 요구사항:
- 성적 입력 폼
  - 시험명 (예: 중간고사, 모의고사)
  - 과목별 점수
  - 시험 날짜
  - 목표 점수 대비 달성률 자동 계산
  
- 일괄 입력 기능
  - CSV 업로드
  - 엑셀 템플릿 제공

#### 3. 시각화 대시보드
기능 설명:
- 학생별 성적 추이 그래프
- 목표 달성도 진행률 표시
- 과목별 강점/약점 분석

차트 종류:
1. 라인 차트: 시간별 성적 추이
2. 레이더 차트: 과목별 균형도
3. 도넛 차트: 목표 달성률
4. 바 차트: 과목별 점수 비교
5. 히트맵: 출석률 캘린더

#### 4. 목표 설정 및 추적
기능 설명:
- 단기/장기 목표 설정
- 목표별 마일스톤 설정
- 달성률 자동 계산

목표 타입:
- 점수 목표: "수학 90점 달성"
- 성적 향상: "지난 대비 10점 향상"
- 출석 목표: "한 달 100% 출석"
- 과제 완료: "과제 제출률 95%"

#### 5. 사용자 관리 (권한)
기능 설명:
- 역할 기반 접근 제어 (RBAC)
- 학원 관리자, 강사, 학부모, 학생 권한 분리

권한 매트릭스:
기능              | 관리자 | 강사 | 학부모 | 학생
학생 등록/수정    |   O    |  O   |   X    |  X
성적 입력        |   O    |  O   |   X    |  X
대시보드 조회    |   O    |  O   |   O    |  O
목표 설정        |   O    |  O   |   △    |  △
리포트 다운로드  |   O    |  O   |   O    |  X

△ = 본인 것만 가능

## 5. 기술 스택

- Framework: Next.js 14 (App Router)
- Language: TypeScript
- UI Library: shadcn/ui
- Styling: Tailwind CSS

상태 관리:
- Server State: TanStack Query (React Query)
- Client State: Zustand
- Form State: React Hook Form + Zod

차트 라이브러리:
- Charts: Recharts / Chart.js / Apache ECharts
- 대안: Tremor (tailwind 기반)

인증:
- Auth: NextAuth.js (v5)
- Session: JWT + Refresh Token

기타:
- Date: date-fns
- Icons: Lucide React
- Toast: Sonner

### 5.2 Backend

API:
- Framework: Next.js API Routes
- ORM: Prisma
- Database: PostgreSQL (Supabase 추천)
- Validation: Zod

파일 저장:
- Storage: AWS S3 / Cloudflare R2

실시간:
- WebSocket: Socket.io (optional)

### 5.3 DevOps

배포:
- Hosting: Vercel (Web)
- Database: Supabase / AWS RDS
- CDN: Cloudflare

CI/CD:
- GitHub Actions
- 테스트: Jest + React Testing Library

모니터링:
- Analytics: Vercel Analytics
- Error Tracking: Sentry
- Logging: Better Stack

### 5.4 Mobile (Phase 2)

- Framework: React Native (Expo)
- State: Redux Toolkit
- Navigation: React Navigation

## 6. 사용자 스토리

### 6.1 학원 원장/강사 스토리

Feature: 학생 성적 관리
As a 학원 원장
I want to 학생의 성적을 간편하게 입력하고
So that 시각화된 리포트를 즉시 확인할 수 있다

Scenario: 신규 학생 등록
Given 내가 대시보드에 로그인되어 있고
When "학생 추가" 버튼을 클릭하고
And 학생 정보를 입력하면 (이름, 학년, 목표 점수)
Then 학생이 시스템에 등록되고
And 해당 학생의 프로필 페이지로 이동한다

Scenario: 시험 성적 입력
Given 학생 "김철수"의 프로필 페이지에 있고
When "성적 추가" 버튼을 클릭하고
And 과목별 점수를 입력하면 (수학: 85, 영어: 90)
Then 성적이 저장되고
And 성적 추이 그래프가 자동으로 업데이트된다

### 6.2 학부모 스토리

Feature: 자녀 성적 모니터링
As a 학부모
I want to 자녀의 학습 현황을 실시간으로 확인하고
So that 효과적으로 학습을 지원할 수 있다

Scenario: 자녀 대시보드 조회
Given 내가 학부모로 로그인되어 있고
When 메인 대시보드에 접속하면
Then 자녀의 최근 성적 그래프가 보이고
And 목표 달성률이 퍼센트로 표시되고
And 출석률이 표시된다

## 8. 데이터 구조

### 8.1 Database Schema (Prisma)

// schema.prisma

model Academy {
  id              String   @id @default(cuid())
  name            String
  address         String?
  phone           String?
  subscriptionPlan String @default("free")
  subscriptionEnd DateTime?
  
  users           User[]
  students        Student[]
  
  createdAt       DateTime @default(now())
  updatedAt       DateTime @updatedAt
}

model User {
  id              String   @id @default(cuid())
  email           String   @unique
  name            String
  role            Role     @default(TEACHER)
  password        String
  
  academyId       String
  academy         Academy  @relation(fields: [academyId], references: [id])
  
  children        Student[] @relation("ParentToStudent")
  studentProfile  Student? @relation("StudentUser")
  
  createdAt       DateTime @default(now())
  updatedAt       DateTime @updatedAt
}

enum Role {
  ADMIN
  TEACHER
  PARENT
  STUDENT
}

model Student {
  id              String   @id @default(cuid())
  name            String
  grade           String
  birthDate       DateTime?
  phone           String?
  
  academyId       String
  academy         Academy  @relation(fields: [academyId], references: [id])
  
  parents         User[]   @relation("ParentToStudent")
  userId          String?  @unique
  user            User?    @relation("StudentUser", fields: [userId], references: [id])
  
  goals           Goal[]
  grades          Grade[]
  attendances     Attendance[]
  
  notes           String?
  
  createdAt       DateTime @default(now())
  updatedAt       DateTime @updatedAt
}

model Goal {
  id              String   @id @default(cuid())
  title           String
  description     String?
  targetValue     Float
  currentValue    Float    @default(0)
  unit            String   @default("점")
  deadline        DateTime?
  status          GoalStatus @default(IN_PROGRESS)
  
  studentId       String
  student         Student  @relation(fields: [studentId], references: [id], onDelete: Cascade)
  
  createdAt       DateTime @default(now())
  updatedAt       DateTime @updatedAt
}

enum GoalStatus {
  IN_PROGRESS
  ACHIEVED
  FAILED
  CANCELLED
}

model Grade {
  id              String   @id @default(cuid())
  examName        String
  examDate        DateTime
  examType        String   @default("정기고사")
  
  subject         String
  score           Float
  maxScore        Float    @default(100)
  
  studentId       String
  student         Student  @relation(fields: [studentId], references: [id], onDelete: Cascade)
  
  memo            String?
  
  createdAt       DateTime @default(now())
  updatedAt       DateTime @updatedAt
}

model Attendance {
  id              String   @id @default(cuid())
  date            DateTime
  status          AttendanceStatus
  
  studentId       String
  student         Student  @relation(fields: [studentId], references: [id], onDelete: Cascade)
  
  memo            String?
  
  createdAt       DateTime @default(now())
  updatedAt       DateTime @updatedAt
  
  @@unique([studentId, date])
}

enum AttendanceStatus {
  PRESENT
  ABSENT
  LATE
  EXCUSED
}

## 11. 성공 지표 (KPIs)

### 11.1 비즈니스 지표

고객 지표:
- 베타 테스터: 10개 학원 (3개월 내)
- 유료 전환율: 30% 이상
- 월간 활성 학원(MAA): 100개 (6개월 내)
- 연간 순환 학원(AAA): 300개 (12개월 내)
- 고객 이탈률(Churn Rate): 5% 이하

매출 지표:
- MRR (월간 반복 매출): 
  - 3개월: ₩5,000,000
  - 6개월: ₩20,000,000
  - 12개월: ₩100,000,000
- ARPU (유저당 평균 매출): ₩70,000
- LTV (고객 생애 가치): ₩2,000,000 이상
- CAC (고객 획득 비용): ₩500,000 이하
- LTV/CAC 비율: 4:1 이상

### 11.2 제품 지표

사용성 지표:
- DAU/MAU 비율: 60% 이상
- 평균 세션 시간: 10분 이상
- 주간 활성 사용자: 전체의 80%
- 기능별 사용률:
  - 성적 입력: 90%
  - 대시보드 조회: 95%
  - 리포트 다운로드: 60%
  - 목표 설정: 70%

만족도 지표:
- NPS (순추천지수): 50 이상
- CSAT (고객 만족도): 4.5/5.0 이상
- 앱스토어 평점: 4.7/5.0 이상

기술 지표:
- 페이지 로드 시간: 2초 이하
- API 응답 시간: 500ms 이하
- 가동 시간(Uptime): 99.9%
- 버그 해결 시간: 24시간 이내

### 11.3 성장 지표

획득 채널:
- 검색 유입: 30%
- 소셜 미디어: 20%
- 추천/입소문: 25%
- 유료 광고: 15%
- 기타: 10%

바이럴 지표:
- K-Factor: 0.5 이상
- 추천율: 학원당 평균 1.5개 추천
- 소셜 공유: 월 100회 이상
